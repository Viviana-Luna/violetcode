# VioletCode 品牌规范

VioletCode 以紫罗兰花作为产品识别符号。图标中心的代码尖括号表达开发工具属性，五瓣轮廓负责在小尺寸和终端环境中保持识别度。

## 核心资产

- `assets/brand/violetcode-icon.svg`：完整彩色产品图标，是所有位图导出的唯一源文件。
- `assets/brand/violetcode-mark-mono.svg`：单色标记，用于小尺寸、印刷和受限配色环境。
- `assets/brand/png/`：16、32、64、128、256、512 与 1024 像素 PNG 导出。
- `assets/brand/VioletCode.iconset/`：macOS 应用图标源尺寸集合。

## 色彩

| 用途 | 深色界面 | 浅色界面 |
| --- | --- | --- |
| 品牌主色 | `#B794FF` | `#5B33B9` |
| 品牌高亮 | `#D3C2FF` | `#7C58D6` |
| 深色背景 | `#08060D` | — |
| 深色表面 | `#171024` | — |

成功、警告和错误必须继续使用独立语义色。紫罗兰只表达品牌、焦点和主要操作，不能作为错误或成功的唯一提示。

## 使用原则

- 不改变花瓣比例、旋转角度或中心尖括号关系。
- 不在复杂照片上直接放置彩色图标；必要时使用单色标记。
- 小于 24 像素时优先使用单色标记，避免依赖中心细节识别。
- 产品内统一写作 `VioletCode`；终端标题因空间需要可写作 `VioletCode`。
- 不再使用旧产品名称、星形或动物形象作为 VioletCode 品牌标识。
