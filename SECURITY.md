# 安全策略

## 报告安全问题

请优先使用 GitHub 仓库的 Private vulnerability reporting / Security Advisory 私下报告安全问题。不要在公开 Issue、日志或截图中提交以下内容：

- DeepSeek、火山方舟、Exa 或 GitHub 凭据
- 完整 `auth.json`
- 会话正文、个人路径或其他敏感数据

报告中请提供受影响版本、最小复现步骤、预期行为、实际影响和已经尝试的缓解方式。维护者确认问题前不承诺固定响应时限。

## 支持范围

当前只维护 `main` 分支最新版本。早期版本可能不会单独回补安全修复。

## 凭据泄露

如果凭据已经出现在提交、Issue、日志或截图中，应立即到对应 Provider 撤销并重新生成；仅删除公开文本不能使已经泄露的凭据重新安全。
